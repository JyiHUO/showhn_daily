**作者：** caleblloyd

**描述：**
CargoWall 是一个轻量级的 eBPF 防火墙，专为 GitHub Actions 设计。它最初是为了阻止 AI 代理访问不受信任的域名而构建，但鉴于近期 GitHub Actions 的安全事件，它也可以有效地阻止来自 CI 运行器的恶意连接。它通过精密的网络流量控制和进程关联，确保只有合法的连接才能通过，大大增强了 CI/CD 流水的安全性。

**如何使用：**
对于开发者来说，使用 CargoWall 非常简单，就像在你的 GitHub Actions 工作流中添加一个新步骤一样。你只需要在你的 YAML 文件中引入 `code-cargo/cargowall-action` 这个 Action。你可以通过 `allowed-hosts` 参数指定允许的域名列表，比如 `registry.npmjs.org`。设置好之后，CargoWall 就会在你的 Actions 运行环境中自动启动，并开始 enforcing（强制执行）你的网络策略。例如，你可以设置 `mode: enforce` 来严格阻止所有非允许的连接，或者在 `monitor` 模式下只记录可疑的连接，以便进行分析。这提供了一个简单直接的方式来加固你的 CI/CD 环境，防止潜在的安全风险。
