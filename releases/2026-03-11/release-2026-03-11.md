### 2.网页元素变更侦测器 (Site Spy)

![网页元素变更侦测器 (Site Spy)](https://showhntoday.com/images/47337607.png)

**URL：** [https://sitespy.app](https://sitespy.app)

**作者：** vkuprin

**描述：**
Site Spy 是一个浏览器扩展和网页仪表盘组合工具，它能帮你密切关注网页的细微变化。它最核心的创新在于，不像其他工具只能监控整个网页，Site Spy 可以让你精准定位网页上的特定元素（比如商品价格、股票状态、新闻标题），一旦这些元素发生变化，它会立刻通知你，并以清晰的“差异对比”视图展示变更内容，还可以将变更历史整理成 RSS 订阅源，方便你随时查阅，告别错过重要信息。所以，它解决了你可能因为网页内容更新而不自知，从而错过关键信息或机会的烦恼。

**如何使用：**
开发者可以使用 Site Spy 作为一款强大的信息监视和自动化流程触发器。首先，你可以通过浏览器扩展（Chrome/Firefox）方便地安装和使用。在你想监控的网页上，打开 Site Spy 扩展，使用它的“元素选择器”精确地框选出你关心的内容区域。然后，你可以选择如何接收通知：可以是浏览器弹出通知、电子邮件，甚至是 Telegram 消息。更进一步，Site Spy 可以将这些变化推送到 RSS 订阅源，这对于需要整合到其他自动化工作流中的开发者来说是极其宝贵的。例如，你可以用 RSS 读取器订阅特定网站上的商品价格变化，当价格低于某个阈值时，自动触发购买脚本；或者监控竞品网站的产品更新，及时获取信息。此外，它还提供了一个 MCP 服务器接口，可以与 Claude、Cursor 等 AI 代理集成，让 AI 能够感知并响应网页内容的实时变化。所以，对开发者而言，Site Spy 就像是为网页信息添加了一个实时感知和响应的“眼睛”，极大地扩展了自动化和信息获取的边界。

### 4.AgentSync-Browser

**如何使用：**
开发者如何使用 AgentSync-Browser？

### 5.nah: 智能指令权限卫士

![nah: 智能指令权限卫士](https://showhntoday.com/images/47343927.png)

**URL：** [https://github.com/manuelschipper/nah/](https://github.com/manuelschipper/nah/)

**作者：** schipperai

**描述：**
nah 是一个创新的 Claude Code（一个可以执行代码的 AI 助手）的权限管理工具。它能智能识别 Claude Code 尝试执行的每一个命令，精确分类其操作类型（例如：读取文件、写入数据库、修改 Git 历史等），然后根据预设的规则决定是否允许、需要用户确认、还是直接拒绝。这比传统的“一刀切”权限管理更精细，能有效防止 AI 助手误操作带来的风险，比如意外删除文件或泄露敏感信息。所以，它能让你更放心地使用 AI 来帮你写代码，而不用担心潜在的安全问题。

**如何使用：**
开发者安装 nah 后，可以通过简单的命令 `pip install nah && nah install` 来完成基本配置。之后，nah 会自动作为 Claude Code 的一个“前置钩子”（PreToolUse hook）。当你使用 Claude Code 并让它尝试执行某个操作时，nah 会先拦截这个命令，进行分析和权限判断。你可以在它工作时，观察它对命令的分类和判断结果。如果遇到不确定的命令，nah 会弹出提示让你确认，并且你还可以将这个判断结果记录下来，下次遇到同类命令时，nah 就能自动处理。对于高级用户，nah 提供了丰富的自定义选项，可以让你根据自己的具体需求，调整权限策略和分类规则。所以，它的使用方式灵活，既能开箱即用，也能深度定制，满足不同开发者的需求，让你能安全地利用 AI 提高开发效率。

**如何使用：**

开发者可以通过Sandbox Flow的可视化界面来构建AI代理。你可以在界面上选择不同的功能模块（节点），例如：'浏览器导航'、'LLM指令执行'、'文件操作'、'API调用'等，然后用连线将它们连接起来，设定数据流动的方向和逻辑。这样，你就可以设计一个AI代理，让它先浏览网页获取信息，然后利用LLM来分析和总结这些信息，最后根据分析结果执行某个操作，比如保存到文件或发送邮件。这种方式非常适合需要AI执行多步骤复杂任务的场景，比如自动化数据收集、内容生成、用户交互模拟等。未来还可以集成本地LLM和更强大的本地沙盒，进一步提升安全性和自主性。
