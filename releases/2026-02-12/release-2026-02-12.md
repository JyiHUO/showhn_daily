### 1.Decision Guardian: 代码中的决策守卫者

**作者：** iamalizaidi

**描述：**
Decision Guardian 是一个 GitHub Actions 工具，它允许你将重要的架构决策记录（ADRs）以 Markdown 格式编写，并与特定的文件路径关联。当有人提交一个修改这些文件的 Pull Request（PR）时，Decision Guardian 会自动在 PR 中发布一条评论，提供相关的上下文信息，例如：当初为什么做出这个决定、有哪些备选方案被否决了。它就像是为代码所有权（CODEOWNERS）的“谁”提供了“为什么”的解决方案，让团队成员更容易理解项目的历史决策，避免重复评估。

**如何使用：**
开发者可以在项目的 GitHub 仓库中创建一个专门的目录（例如 `docs/adr`）来存放决策记录文件，这些文件使用 Markdown 格式编写，并按照预设的结构（例如，使用特定的标题或标签来指明决策内容、理由、备选方案等）来记录。然后，在 GitHub Actions 工作流中配置 Decision Guardian，让它在你提交 Pull Request 时自动运行。当你的 PR 修改了与某个决策记录文件路径相关的代码文件时，Decision Guardian 就会自动扫描这个 PR 的更改，找到最相关的决策记录，并将决策的关键信息以评论的形式发布到你的 PR 页面上。这样，任何参与代码评审的团队成员，即使不熟悉这个决策的历史，也能快速了解到相关的背景信息，从而做出更明智的判断。你可以把它看作是给你的代码仓库注入了一份“智能决策百科全书”，在代码变动时自动为你提供参考。

### 5.BetterDB - 永恒的数据库观察者

**作者：** kaliades

1.  **通过 npm 包直接运行：** 如果你已经安装了 Node.js 环境，可以直接在终端输入 `npx @betterdb/monitor` 命令。这会立即启动 BetterDB 的监控界面，你就可以开始连接你的 Valkey 或 Redis 实例进行监控了。
2.  **通过 Docker 容器运行：** 如果你更喜欢使用 Docker，可以运行 `docker run -d -p 3001:3001 betterdb/monitor` 命令。这会在后台启动一个 BetterDB 的容器，并将监控界面暴露到你的本地 3001 端口。这样，即使你没有 Node.js 环境，也能轻松启动并使用它。

### 7.Yori - AI代码语义隔离容器

**作者：** alonsovm

**描述：**
Yori是一个创新的工具，它通过引入“语义容器”的概念，解决了AI代码助手过度修改代码的问题。它允许开发者将AI生成代码的部分（容器）与自己手动编写的稳定代码（宿主）明确分开，确保AI只能在开发者指定的范围内进行操作，从而提高了AI在开发过程中的安全性、可控性和可移植性。

**如何使用：**
开发者可以将Yori的特殊语法嵌入到现有代码文件中。例如，在一个C++文件里，你可以将需要AI协助实现的功能放在$$${...}$$代码块内，并在外部声明需要导出的文件（如 `EXPORT: "myfile.cpp"`）。当你运行Yori命令行工具（如 `yori myutils.md -make -series`）时，Yori会读取这个文件，将外部的宿主代码原封不动地复制，然后将$$${...}$$内的自然语言指令发送给本地或云端的大型语言模型（LLM），由LLM生成代码填充到容器中。最后，Yori会使用你本地的编译工具链（如GCC、Clang或Python）编译生成最终代码。如果编译失败，Yori还会把错误信息反馈给LLM，让它重新尝试修复。这对于希望利用AI提升开发效率，但又担心AI过度干预代码的开发者来说非常有用。

### 10.PolyMCP-WASM: 浏览器里的Python工具箱

**作者：** justvugg

**描述：**
PolyMCP-WASM 允许你把你用 Python 写的工具（例如处理数据的函数、计算器等）直接编译成 WebAssembly (WASM)。这意味着这些 Python 工具不再需要一个单独的服务器来运行，可以直接在浏览器、Node.js 或者边缘计算环境中运行，并且完全独立、安全。你可以把你常用的 Python 工具打包成一个 WASM 文件，然后让AI智能体或者其他应用直接调用，就像调用一个本地函数一样方便。这让构建更强大、更灵活的AI应用变得更容易。

**如何使用：**
开发者可以将现有的 Python 工具函数，通过 PolyMCP-WASM 的 `expose_tools_wasm` 函数进行标记，然后使用 `compiler.compile()` 命令将其编译成 WASM 文件。这个生成的文件可以直接部署到 Web 服务器上，然后在前端 JavaScript 代码中加载并调用。例如，你可以在你的网页中引入这个 WASM 文件，然后直接调用之前编写的 Python 函数，比如一个计算、数据格式转换或者文本处理函数。它还可以集成到 Node.js 环境，或者部署到边缘计算平台，让这些环境也能直接运行你的 Python 工具。
