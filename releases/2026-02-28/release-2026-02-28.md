### 2.PowerShell 窗格魔术师 (Psmux)

**作者：** uniquegodwin

**描述：**
Psmux 是一个能在 Windows PowerShell 中模拟 tmux 工作流的工具，而无需依赖 WSL、Cygwin 或 MSYS2。它用 Rust 编写，可以读取你熟悉的 .tmux.conf 配置，并支持分屏、会话管理、鼠标模式以及常用的 tmux 快捷键。它的创新在于将 Linux 终端强大的分屏协作能力原生带入 Windows，让 Windows 用户也能享受高效的命令行操作。

**如何使用：**
开发者可以通过下载并安装 Psmux，然后直接在 PowerShell 中启动它。你可以在 PowerShell 中输入 `psmux` 命令来创建一个新的会话，然后使用 `psmux split-h` (水平分割) 或 `psmux split-v` (垂直分割) 来创建新的窗格。你可以通过熟悉的 tmux 快捷键（例如 Ctrl+B 后面跟方向键）在窗格之间切换。如果你有 .tmux.conf 文件，可以直接放在 Psmux 的配置目录下，它会自动加载，帮你恢复你习惯的界面布局和快捷键设置。所以这对我有什么用？你可以把 Psmux 想象成一个帮你把命令行操作“打包”起来的工具，当你需要同时运行多个命令、监控日志、或者执行一系列自动化脚本时，Psmux 就能让你在一个屏幕上全部搞定，非常方便。

### 6.Vigil: AI Agent 的安全管家

**作者：** HexitLabs

**描述：**
Vigil 是一个零依赖、纯粹基于规则的AI智能体工具调用检查引擎。它能在AI智能体执行命令前，像一个忠诚的守卫一样，检查这些命令是否安全，有效防止AI智能体误执行危险操作，比如删除重要文件或泄露敏感信息。它的核心价值在于用一种确定性的、非AI的方式来保障AI系统的安全。

**如何使用：**
开发者可以将Vigil集成到他们的AI应用中，通常是在AI智能体调用工具函数之前。举个例子，如果你的AI应用使用了LangChain框架，你可以将Vigil接入到LangChain的工具链（Toolchain）中。如果你的应用是基于Node.js开发的，可以直接通过npm安装Vigil库，然后在代码中引入`checkAction`函数，传入AI将要执行的操作，Vigil就会返回一个结果，告诉你是允许（ALLOW）还是阻止（BLOCK）这个操作，并说明原因。这样，即使AI智能体自己“犯糊涂”想做危险的事情，Vigil也能及时制止，保护你的系统和数据安全。

### 7.ClaudeCode时光追踪CLI

### 9.Coi-跨平台混合渲染语言

**作者：** io_eric

**描述：**
Coi 是一个创新的类型安全语言，它能编译成 WebAssembly，旨在实现跨平台（Web, 移动端, 桌面端, 服务器）的开发，同时避免传统跨平台框架（如 Flutter）在 Web 端出现的体积膨胀问题。它通过在 Web 端利用浏览器原生的 HTML/CSS/JS，而在桌面和移动端使用 C++/Skia 来渲染 UI，从而实现‘混合渲染’，兼顾了代码复用和性能优化。

**如何使用：**
开发者可以学习 Coi 语言，用它来编写应用逻辑。在 Web 开发场景下，Coi 会被编译成 WebAssembly，并与原生的 HTML/CSS/JS 结合，运行在浏览器中。这意味着你可以用 Coi 来控制网页的交互和数据处理，但网页的视觉呈现依旧由浏览器强大的渲染引擎负责，不会出现‘canvas 爆炸’的问题。对于移动端和桌面端，Coi 则会通过 C++/Skia 的技术栈进行编译和渲染，生成原生应用。开发者可以想象成用一种语言写完代码后，它会根据目标平台，自动选择最合适的‘画图’和‘运行’方式。

### 11.AI计划审查官

**作者：** yuu1ch13

**描述：**
这个项目巧妙地利用了AI写代码前会先制定详细计划的特性。它创建了一个“教练”AI，在AI（比如Claude）生成执行计划后，会把这个计划发送给另一个“评审”AI（如Codex或Gemini）来挑毛病。评审AI会找出计划里的不足之处，然后把反馈给原AI，让它修改计划。这个过程就像是给AI写计划增加了“同行评审”，确保计划更完善。最终，你可以看到计划的修改过程，了解AI是如何改进的。

### 12.自进化文件系统（O-O）

**作者：** jahala

**描述：**
O-O 是一个开创性的项目，它能让 HTML 和 Bash 脚本文件互相学习、互相修改，实现代码的自我进化。这次更新的核心亮点是实现了文件之间的同步：你只需要在一个文件中修改 CSS 样式，然后运行一个简单的命令，所有同级的文件都会自动更新。它完全不需要复杂的构建工具或共享导入，而是通过文件内部的特殊标记来复制和粘贴代码片段，实现了“零依赖，自己动手”的黑客精神。这就像是让你的代码拥有了生命，能够根据你的修改自动适应和传播。

**如何使用：**
开发者可以将 O-O 项目用作一种非常灵活和轻量级的开发模式。想象一下，你正在开发一个简单的网站，里面有多个 HTML 文件和对应的 CSS 文件，负责不同的页面或者组件。以往，你修改了一个 CSS 文件，可能需要运行一个构建命令，或者手动去复制粘贴到其他 CSS 文件中。使用 O-O，你只需要把 HTML 和 CSS 代码写在同一个 O-O 文件里，然后当你想统一修改某个 CSS 属性时，比如给所有的按钮加上暗黑模式的背景色，你只需要在一个文件中编辑好 CSS 部分，然后运行 `bash your_file.o-o --sync css`。这样，你目录下的所有 `.o-o` 文件都会自动接收到这个 CSS 更新。它特别适合于快速原型开发、个人项目、小型静态网站的维护，以及任何你希望减少构建步骤、简化协作流程的场景。集成方式非常简单，你只需要准备好 Bash 环境，然后就可以开始编写 `.o-o` 文件了。
