### 2.Turbolite: S3原生 SQLite 极速冷查询引擎

![Turbolite: S3原生 SQLite 极速冷查询引擎](https://showhntoday.com/images/47534283.png)

**URL：** [https://github.com/russellromney/turbolite](https://github.com/russellromney/turbolite)

**作者：** russellthehippo

**描述：**
Turbolite 是一个用 Rust 编写的 SQLite VFS (Virtual File System)，它能直接从 Amazon S3 对象存储服务中以亚秒级甚至更快的速度执行冷查询。它通过智能地解析 SQLite 的 B-tree 结构，将相关数据页打包成压缩组，并利用 S3 的 Range GET 请求，实现高效的数据检索，解决了在云存储上运行嵌入式数据库时，因网络延迟和存储特性不匹配而导致的性能瓶颈。

**如何使用：**
开发者可以将 Turbolite 集成到现有的 Rust 项目中。当需要使用 SQLite 数据库时，不是直接创建一个本地文件，而是配置 Turbolite 连接到你的 S3 存储桶。这样，所有对 SQLite 数据库的操作都会通过 Turbolite 转发到 S3。它特别适合那些拥有大量“冷”数据库（即数据很少被实时修改，但需要快速检索）的场景，例如多租户应用中每个租户一个数据库，或者会话级别的临时数据库。通过调整 Turbolite 的预取策略，你可以平衡查询速度和资源消耗，让点查询、小范围 JOIN 操作和大型数据扫描都能获得更好的性能。
