### 1.NanoClaw：隔离执行的智能助手框架

![NanoClaw：隔离执行的智能助手框架](https://showhntoday.com/images/46850205.png)

**URL：** [https://github.com/gavrielc/nanoclaw](https://github.com/gavrielc/nanoclaw)

**作者：** jimminyx

**描述：**
NanoClaw 是一个精简的 TypeScript 框架，仅用约 500 行核心代码，通过利用 Apple 容器技术，实现了智能助手（Agent）的沙盒隔离执行。它解决了传统大型框架中 Agent 权限过大、资源互相干扰的安全和稳定性问题，为开发者提供了一个安全、独立的 Agent 运行环境。这对于需要构建多 Agent 系统，但又担心它们相互影响或滥用权限的开发者来说，是一个非常有价值的创新。

**如何使用：**
开发者可以使用 NanoClaw 来快速搭建和部署需要隔离运行的智能助手。其核心在于利用 Apple 容器技术，为每个 Agent 进程创建一个独立的运行环境。这意味着每个 Agent 都有自己的文件系统、进程空间，与其他 Agent 完全隔离。你可以将 NanoClaw 集成到你的 Node.js 项目中，编写你的 Agent 逻辑，然后通过 NanoClaw 的接口启动它们。例如，你可以为不同的客户服务场景、数据处理任务或者自动化流程构建独立的 Agent，并确保它们之间不会因为权限问题或资源冲突而产生bug。这样做的好处是，即使某个 Agent 出错了，也不会影响到整个系统的运行，而且数据安全也更有保障。

### 8.nono: 内核级AI代理代码隔离器

![nono: 内核级AI代理代码隔离器](https://showhntoday.com/images/46849615.png)

**URL：** [https://nono.sh](https://nono.sh)

**作者：** decodebytes

**描述：**
nono是一个为AI代理设计的安全沙箱工具，它利用操作系统底层的安全机制（Linux的Landlock和macOS的Seatbelt）来严格限制AI代理执行代码时的权限。这解决了AI代理可能通过提示注入、幻觉或工具漏洞来访问敏感文件（如SSH密钥）或进行数据泄露的风险。nono通过精细的文件系统访问控制、网络隔离以及安全地管理和注入敏感信息，为AI代理提供了一个无法逃逸的安全执行环境。

**如何使用：**
开发者可以使用nono来安全地运行任何需要执行代码的AI代理。比如，如果你正在使用一个AI助手来帮你写代码，你可以用nono来运行它：`nono run --read ./src --allow ./output -- cargo build`。这句话的意思是，允许AI代理读取当前目录下的`src`文件夹（比如你的源代码），并且只允许它把编译生成的文件放在`output`文件夹里，然后执行`cargo build`命令（一个Rust项目的构建命令）。这样做的好处是，即使AI代理有恶意行为，它也无法读取你电脑上其他地方的文件，也无法把不该放的文件放到其他地方。nono还可以用来执行其他类型的AI任务，例如：`nono run --profile claude-code -- claude`（运行一个名为claude的AI助手，并应用特定的安全配置），`nono run --allow . --net-block -- npm install`（允许AI代理在当前目录下操作，但完全阻止它访问网络，然后执行npm install命令，比如安装Node.js的包），或者`nono run --secrets api_key -- ./my-agent`（运行一个AI代理，并安全地注入名为`api_key`的密钥，执行完后会自动清除）。
